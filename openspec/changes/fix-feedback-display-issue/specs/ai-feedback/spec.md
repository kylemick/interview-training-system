## MODIFIED Requirements

### Requirement: 反馈格式和展示
係統SHALL以清晰、易读的格式展示反馈內容，并確保所有類別的反馈數據都能正確解析和显示。

#### Scenario: 結构化反馈展示
- **WHEN** 用户查看单題反馈
- **THEN** 按固定結构展示：語言质量、內容深度、具体建議、參考答案
- **AND** 使用評分、標籤、重點標注等方式增强可读性
- **AND** 係統必须正確处理 JSON 格式的反馈數據（无論是字符串还是對象）
- **AND** 当反馈數據格式异常時，显示友好的错误提示而不是空白页面

#### Scenario: 所有類別的反馈正確显示
- **WHEN** 用户查看任何類別（包括 logic-thinking 和 current-affairs）的反馈
- **THEN** 反馈內容必须完整显示，包括評分、優點、待改進、建議、參考思路和參考答案
- **AND** 係統必须統一处理類別名称（logic-thinking 和 logical-thinking 视为同一類別）
- **AND** 後端返回的反馈數據必须是已解析的 JSON 對象，而不是字符串
- **AND** 前端必须添加防御性解析邏輯，处理可能的格式异常情况

#### Scenario: 反馈數據解析容错
- **WHEN** 反馈數據在傳输或存储過程中格式异常
- **THEN** 係統必须尝試解析 JSON 字符串（如果接收到的是字符串）
- **AND** 如果解析失敗，記錄错误日志但不導致页面崩溃
- **AND** 显示友好的错误提示，允许用户重新生成反馈

#### Scenario: 會話總結报告
- **WHEN** 用户查看完整會話的總結
- **THEN** 展示：整体得分、各維度得分、弱點列表、進步對比、下次建議
- **AND** 支持導出为PDF或文本格式
- **AND** 確保所有反馈數據在總結中正確显示
