# Change: 创建弱点管理页面

## Why

当前系统中，弱点分析功能的后端API已完整实现，弱点数据在Dashboard、Progress、Settings等页面有部分展示，但缺少一个专门的弱点管理页面。这导致：

1. 用户无法集中查看和管理所有弱点记录
2. 弱点详情查看、状态更新、删除等操作分散在不同页面，操作不便
3. 无法对弱点进行批量操作和高级筛选
4. 弱点统计和可视化功能不完整，用户难以全面了解弱点改善情况
5. 缺少弱点详情页面，无法查看弱点的完整信息（改进建议、相关话题、练习历史等）

## What Changes

### 核心功能
- 创建独立的弱点管理页面 (`/weaknesses`)，提供完整的弱点CRUD操作
- 弱点列表页面：支持多维度筛选（类别、状态、严重程度、弱点类型）、排序、分页
- 弱点详情页面：展示完整信息（描述、示例、改进建议、相关话题、练习历史、状态变更记录）
- 弱点状态管理：支持更新状态（active/improved/resolved）、删除弱点
- 批量操作：支持批量更新状态、批量删除、批量生成针对性题目
- 弱点统计卡片：总弱点数、按状态分布、按严重程度分布、按类别分布
- 弱点可视化：趋势图表、类型分布图、改善率分析
- 快速操作：一键生成针对性题目、查看相关练习记录

### 用户体验增强
- 在导航菜单中添加"弱点管理"入口（使用WarningOutlined图标）
- 弱点列表支持表格和卡片两种视图模式
- 提供筛选预设（如"高严重度活跃弱点"、"已改善弱点"等）
- 详情页面支持编辑弱点的描述和改进建议
- 状态变更时显示确认对话框，避免误操作

### 技术实现
- 前端新增页面：`frontend/src/pages/Weaknesses/index.tsx`
- 前端新增路由：`/weaknesses` 和 `/weaknesses/:id`
- 复用现有API：`/api/weaknesses` 系列接口（无需新增后端接口）
- 导航菜单更新：在Layout组件中添加弱点管理菜单项

**非BREAKING**：这是新增功能，不影响现有功能。现有页面的弱点展示功能保持不变。

## Impact

### 影响规范
- **weakness-management** (新增): 定义弱点管理页面的功能和交互规范

### 影响代码
- **前端**:
  - `App.tsx`: 添加 `/weaknesses` 和 `/weaknesses/:id` 路由
  - `components/Layout.tsx`: 添加弱点管理菜单项
  - `pages/Weaknesses/index.tsx`: 新建弱点管理页面（列表+详情）
- **后端**:
  - 无需修改，复用现有 `/api/weaknesses` 接口

### 实现状态
- 📋 **待实现**: 所有功能待开发

## Benefits

1. **集中管理**: 用户可以在一个页面查看和管理所有弱点，提高操作效率
2. **完整信息**: 详情页面展示弱点的完整信息，帮助用户更好地理解问题和改进方向
3. **数据驱动**: 统计和可视化帮助用户了解弱点改善趋势，制定针对性训练计划
4. **操作便捷**: 批量操作和快速筛选提高管理效率
5. **用户体验**: 统一的界面设计，与系统其他页面保持一致
