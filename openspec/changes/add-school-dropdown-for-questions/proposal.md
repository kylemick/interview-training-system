# Change: 题库创建使用数据库学校下拉列表

## Why

目前题库创建（手工录入和AI生成）时,学校字段(school_code)需要用户手动输入文本,存在以下问题:
1. 用户可能输入错误的学校代码,导致数据不一致
2. 无法提供标准化的学校选择,影响数据质量
3. 用户体验不好,需要记住学校代码

学校档案表(school_profiles)已经存储了所有学校信息,应该通过下拉列表的方式供用户选择,确保数据一致性和用户体验。

## What Changes

- 题库管理前端页面添加学校下拉列表组件,从数据库动态加载学校列表
- AI生成题目界面添加学校下拉列表组件
- 手动添加/编辑题目界面的学校字段从文本输入改为下拉选择
- 确保学校列表实时从`school_profiles`表获取
- 保留空值选项,允许不指定目标学校

## Impact

- 受影响的规格: `question-bank` (题库管理)
- 受影响的代码:
  - 前端: `frontend/src/pages/Questions/index.tsx` (题库管理页面)
  - 后端: 已有 `/api/schools` 接口可直接使用
- 数据库: 无需修改,已有外键约束 `questions.school_code -> school_profiles.code`
- 用户体验: 提升数据一致性和用户操作便利性
