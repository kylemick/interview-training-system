# Change: 修复面試回憶页面并增强問題類型分類

## Why

1. **面試回憶页面无法打開**：前端代碼中使用了未定义的 `SCHOOLS` 变量，導致页面渲染失敗
2. **問題類型分類不够智能**：当前係統依赖用户手動選擇或AI一次性識別，缺乏持续學習和優化机制
3. **历史數據關聯不足**：当問題類型分類規則更新時，历史數據无法自動關聯更新，導致數據不一致
4. **訓練專項識別不持续**：无法根據积累的回憶和反馈持续識別後续訓練專項，影响訓練計劃的針對性

## What Changes

- **修复面試回憶页面**：修复 `SCHOOLS` 未定义問題，使用從API動態加载的學校列表
- **AI自動識別問題類型**：增强AI分類能力，支持基于历史數據持续優化分類準確性
- **定期更新归纳机制**：实现定期批量更新問題類型分類，確保历史數據与新分類規則保持一致
- **历史數據關聯更新**：当分類規則更新時，自動更新所有相關历史記錄（題目、會話、反馈等）
- **持续專項識別**：基于面試回憶和反馈數據，持续識別學生需要加强的訓練專項，并自動推荐針對性練習

## Impact

- **受影响的能力**：
  - `interview-memory`（面試回憶）- 新增能力規范
  - `question-bank`（題庫管理）- 增强分類和更新机制
- **受影响的代碼**：
  - `frontend/src/pages/InterviewMemory/index.tsx` - 修复 SCHOOLS 未定义
  - `backend/src/routes/ai.ts` - 增强分類和更新API
  - `backend/src/db/schema.sql` - 可能需要添加分類历史表
  - `backend/src/ai/` - 新增或增强分類服務
- **數據庫变更**：
  - 可能需要新增 `question_category_history` 表記錄分類变更历史
  - 可能需要新增 `category_classification_rules` 表存储AI分類規則
