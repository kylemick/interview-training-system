# Change: 更新项目文档

## Why

当前 `openspec/project.md` 文档与项目实际实现状态不一致：
- 文档中的功能模块列表不完整，未反映已实现的所有能力
- 技术栈信息需要更新以匹配实际使用的技术
- 项目结构描述需要与实际代码结构对齐
- 缺少对已归档变更和当前能力的完整说明
- 需要反映MySQL访问规范、性能优化、计划-练习联动等已实现的改进

## What Changes

- **更新项目功能模块列表**：反映所有已实现的能力（训练计划、面试练习、AI反馈、题库管理、进度追踪、学校档案、系统设置、计划-练习联动）
- **更新技术栈信息**：确认实际使用的技术（React 18 + TypeScript + Ant Design, Node.js + Express + TypeScript, MySQL 8.0）
- **更新项目结构**：与实际代码目录结构对齐
- **更新能力列表**：列出所有已归档到specs的能力和待归档的能力
- **更新开发规范**：添加MySQL访问规范、性能要求、文档更新要求等
- **更新约束条件**：反映已实现的性能优化、英文题库语言规范等

## Impact

- 受影响文档：`openspec/project.md`
- 新增spec：`specs/documentation/spec.md` - 项目文档管理规范
- 帮助开发者快速了解项目当前状态和已实现功能
- 确保后续开发基于准确的文档信息
