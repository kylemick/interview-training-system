# Capability: School Profiles (學校特征管理)

## ADDED Requirements

### Requirement: 學校特征數據模型
係統SHALL定义標準化的學校特征數據結构。

#### Scenario: 存储學校基本信息
- **WHEN** 係統初始化或添加學校特征
- **THEN** 每所學校包含：學校代碼（如"SPCC"）、全称、简称、學校類型（男校/女校/男女校）
- **AND** 包含面試形式描述（如"个人面試+小組討論"）
- **AND** 包含官网链接和聯係信息（可選）

#### Scenario: 存储面試特征
- **WHEN** 係統記錄學校的面試特點
- **THEN** 包含重點考查內容列表（如["英文口語", "批判性思維", "国际视野"]）
- **AND** 包含面試形式和流程描述
- **AND** 包含面試時長和常见題型
- **AND** 包含評分標準和权重（如"英文占40%，邏輯占30%"）

### Requirement: TOP學校數據錄入
係統SHALL预置香港顶尖中學的面試特征數據。

#### Scenario: 錄入SPCC特征
- **WHEN** 係統初始化時
- **THEN** 包含SPCC（聖保羅男女中學）的完整特征：
  - 重视英文表達和国际视野
  - 强調批判性思維和多元觀點
  - 注重科學素養和STEM教育
  - 面試形式：个人面試（英文为主）+ 小組討論
  - 常见話題：時事、社會議題、科學常識、个人经历与反思

#### Scenario: 錄入QC特征
- **WHEN** 係統初始化時
- **THEN** 包含QC（皇仁書院）的完整特征：
  - 重视學術能力和邏輯思維
  - 强調理性分析和問題解决能力
  - 面試形式：个人面試 + 可能有笔試
  - 常见話題：學術兴趣、邏輯推理、科學常識

#### Scenario: 錄入LSC特征
- **WHEN** 係統初始化時
- **THEN** 包含LSC（喇沙書院）的完整特征：
  - 重视全人發展和領導力
  - 强調团队合作和沟通能力
  - 面試形式：个人面試 + 小組活動
  - 常见話題：课外活動、团队经验、道德判断

#### Scenario: 预置3-5所TOP學校
- **WHEN** 係統初始化完成
- **THEN** 至少包含SPCC、QC、LSC三所學校的完整特征數據
- **AND** 可選包含其他TOP學校（如DGS、DBS等）
- **AND** 所有數據经過验证和格式化

### Requirement: 學校特征查询
係統SHALL提供便捷的學校特征查询功能。

#### Scenario: 按代碼查询學校
- **WHEN** 用户通過學校代碼（如"SPCC"）查询
- **THEN** 返回该學校的完整特征信息
- **AND** 以結构化格式展示（CLI表格或Web卡片）

#### Scenario: 列出所有學校
- **WHEN** 用户请求查看所有支持的學校
- **THEN** 列出學校代碼、全称、简短描述
- **AND** 按字母顺序或热门程度排序

#### Scenario: 搜索學校
- **WHEN** 用户输入學校名称關键詞（如"保羅"）
- **THEN** 返回名称匹配的學校列表
- **AND** 支持中文和英文搜索

### Requirement: 學校特征应用
係統SHALL在計劃生成和反馈中应用學校特征數據。

#### Scenario: 生成計劃時应用學校特征
- **WHEN** AI生成訓練計劃時用户指定了目標學校
- **THEN** 係統将该學校的重點考查內容作为上下文傳递给AI
- **AND** AI生成的計劃倾斜到该學校重视的類別（如SPCC增加英文和時事比例）
- **AND** 在計劃說明中注明針對性調整

#### Scenario: 生成題目時应用學校特征
- **WHEN** AI生成題目時關聯了目標學校
- **THEN** 係統在提示詞中包含该學校的常见話題和風格
- **AND** 生成的題目更符合该校面試特點
- **AND** 題目標記为该學校相關

#### Scenario: 反馈中提供學校針對性建議
- **WHEN** AI生成反馈時知道目標學校
- **THEN** 反馈中包含"針對[學校]建議"部分
- **AND** 指出回答如何更好地匹配该校期望
- **AND** 举例该校面試官可能的關注點

### Requirement: 學校特征管理
係統SHALL支持更新和补充學校特征數據。

#### Scenario: 添加新學校
- **WHEN** 用户添加係統未预置的學校
- **THEN** 通過CLI或配置文件输入學校代碼、名称和特征
- **AND** 係統验证必填字段并保存
- **AND** 新學校立即可用于計劃生成

#### Scenario: 更新學校特征
- **WHEN** 用户發现學校特征數據過時或不準確
- **THEN** 用户可编輯學校的任意字段
- **AND** 係統保存修改历史
- **AND** 更新後的特征应用于新創建的計劃和題目

#### Scenario: 標記學校數據來源
- **WHEN** 學校特征數據被錄入或更新
- **THEN** 記錄數據來源（如"官网資料"、"用户反馈"、"AI推断"）
- **AND** 記錄更新時間和更新人
- **AND** 低置信度數據標注"待验证"

### Requirement: 學校對比功能
係統SHALL支持對比多所學校的面試特征。

#### Scenario: 對比两所學校
- **WHEN** 用户请求對比两所學校（如"SPCC vs QC"）
- **THEN** 并排展示两校的關键特征：
  - 重點考查內容對比
  - 面試形式差异
  - 适合的準備策略差异
- **AND** 高亮显示主要区別

#### Scenario: 選擇學校建議
- **WHEN** 用户查看多所學校對比
- **THEN** 係統根據學生历史表现（如英文强、邏輯弱）
- **AND** 给出哪些學校更匹配學生優勢的建議
- **AND** 提示需要針對性加强的方面
