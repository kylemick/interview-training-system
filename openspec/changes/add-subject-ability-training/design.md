## Context

香港升中面试不仅考查口语表达和思维能力，还重点考查学生的学科基础能力。目前系统支持的七大专项类别主要关注面试技巧和表达能力，但缺少针对学科基础能力的专项训练。

用户需要针对性的学科能力训练：
- 中文阅读理解：通过阅读文章，考察阅读理解、字词理解、观点提炼等能力
- 英文阅读理解：通过阅读英文文章，考察阅读理解、词汇、观点分析等能力
- 数学基础：考察计算能力、数学概念理解、基础数学知识应用
- 科学实践：考察科学现象说明、科学推理、科学行为等能力

## Goals / Non-Goals

### Goals
- 添加四个学科能力类别，支持专门的题目生成和练习
- 保持与现有七大专项类别的兼容性
- 支持在训练计划中包含学科能力训练
- 提供针对性的练习界面和反馈

### Non-Goals
- 不修改现有七大专项类别的定义和行为
- 不改变数据库结构（category字段已支持任意字符串）
- 不强制所有训练计划必须包含学科能力类别（可选功能）

## Decisions

### Decision: 使用新的类别代码而非子类别
- **选择**：添加四个新的类别代码（`chinese-reading`、`english-reading`、`mathematics`、`science-practice`）
- **原因**：
  - 与现有类别系统保持一致
  - 数据库无需修改（category字段已支持）
  - 便于查询、筛选和统计
  - 前端界面可以统一处理

### Decision: 学科能力类别作为可选功能
- **选择**：学科能力类别作为补充，不强制所有训练计划包含
- **原因**：
  - 保持向后兼容性
  - 用户可以根据需要选择是否使用
  - 不影响现有训练计划

### Decision: 阅读理解题目包含文章和问题
- **选择**：中文和英文阅读理解题目包含完整的文章内容（200-500字）和多个问题
- **原因**：
  - 符合实际面试场景
  - 提供完整的阅读体验
  - 便于评估阅读理解能力

### Decision: 英文阅读理解使用全英文
- **选择**：英文阅读理解类别的所有内容（题目、文章、问题、反馈）使用英文
- **原因**：
  - 与 `english-oral` 类别保持一致的语言规范
  - 符合英文阅读理解的训练目标
  - 提升学生的英文阅读能力

### Decision: 题目生成使用专门的提示词
- **选择**：为每个学科能力类别设计专门的AI生成提示词
- **原因**：
  - 不同类别需要不同的生成策略
  - 阅读理解需要生成文章+问题
  - 数学和科学需要生成特定类型的题目
  - 提高生成题目的质量和针对性

## Risks / Trade-offs

### Risk: 类别过多可能导致界面复杂
- **缓解**：使用分类或分组显示，将七大专项和四个学科能力类别分开展示

### Risk: 题目生成质量可能不一致
- **缓解**：为每个类别设计详细的生成提示词，包含示例和要求

### Risk: 阅读理解题目的文章长度可能影响显示
- **缓解**：前端实现滚动查看，支持返回查看文章

### Trade-off: 是否将学科能力类别与相关专项类别关联
- **选择**：作为独立类别，但可以在训练计划中建议关联
- **原因**：保持灵活性，用户可以选择是否关联

## Migration Plan

### 数据迁移
- 无需数据迁移，数据库结构无需修改
- 现有题目和训练计划不受影响

### 代码迁移
- 更新类别定义和类型
- 更新相关查询和生成逻辑
- 更新前端界面

### 向后兼容
- 现有七大专项类别保持不变
- 现有训练计划和练习会话不受影响
- 新功能作为可选功能添加
