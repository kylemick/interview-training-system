# Change: 修复面试回忆页面并增强问题类型分类

## Why

1. **面试回忆页面无法打开**：前端代码中使用了未定义的 `SCHOOLS` 变量，导致页面渲染失败
2. **问题类型分类不够智能**：当前系统依赖用户手动选择或AI一次性识别，缺乏持续学习和优化机制
3. **历史数据关联不足**：当问题类型分类规则更新时，历史数据无法自动关联更新，导致数据不一致
4. **训练专项识别不持续**：无法根据积累的回忆和反馈持续识别后续训练专项，影响训练计划的针对性

## What Changes

- **修复面试回忆页面**：修复 `SCHOOLS` 未定义问题，使用从API动态加载的学校列表
- **AI自动识别问题类型**：增强AI分类能力，支持基于历史数据持续优化分类准确性
- **定期更新归纳机制**：实现定期批量更新问题类型分类，确保历史数据与新分类规则保持一致
- **历史数据关联更新**：当分类规则更新时，自动更新所有相关历史记录（题目、会话、反馈等）
- **持续专项识别**：基于面试回忆和反馈数据，持续识别学生需要加强的训练专项，并自动推荐针对性练习

## Impact

- **受影响的能力**：
  - `interview-memory`（面试回忆）- 新增能力规范
  - `question-bank`（题库管理）- 增强分类和更新机制
- **受影响的代码**：
  - `frontend/src/pages/InterviewMemory/index.tsx` - 修复 SCHOOLS 未定义
  - `backend/src/routes/ai.ts` - 增强分类和更新API
  - `backend/src/db/schema.sql` - 可能需要添加分类历史表
  - `backend/src/ai/` - 新增或增强分类服务
- **数据库变更**：
  - 可能需要新增 `question_category_history` 表记录分类变更历史
  - 可能需要新增 `category_classification_rules` 表存储AI分类规则
