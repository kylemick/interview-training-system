# Capability: School Profiles (学校特征管理)

## ADDED Requirements

### Requirement: 学校特征数据模型
系统SHALL定义标准化的学校特征数据结构。

#### Scenario: 存储学校基本信息
- **WHEN** 系统初始化或添加学校特征
- **THEN** 每所学校包含：学校代码（如"SPCC"）、全称、简称、学校类型（男校/女校/男女校）
- **AND** 包含面试形式描述（如"个人面试+小组讨论"）
- **AND** 包含官网链接和联系信息（可选）

#### Scenario: 存储面试特征
- **WHEN** 系统记录学校的面试特点
- **THEN** 包含重点考查内容列表（如["英文口语", "批判性思维", "国际视野"]）
- **AND** 包含面试形式和流程描述
- **AND** 包含面试时长和常见题型
- **AND** 包含评分标准和权重（如"英文占40%，逻辑占30%"）

### Requirement: TOP学校数据录入
系统SHALL预置香港顶尖中学的面试特征数据。

#### Scenario: 录入SPCC特征
- **WHEN** 系统初始化时
- **THEN** 包含SPCC（圣保罗男女中学）的完整特征：
  - 重视英文表达和国际视野
  - 强调批判性思维和多元观点
  - 注重科学素养和STEM教育
  - 面试形式：个人面试（英文为主）+ 小组讨论
  - 常见话题：时事、社会议题、科学常识、个人经历与反思

#### Scenario: 录入QC特征
- **WHEN** 系统初始化时
- **THEN** 包含QC（皇仁书院）的完整特征：
  - 重视学术能力和逻辑思维
  - 强调理性分析和问题解决能力
  - 面试形式：个人面试 + 可能有笔试
  - 常见话题：学术兴趣、逻辑推理、科学常识

#### Scenario: 录入LSC特征
- **WHEN** 系统初始化时
- **THEN** 包含LSC（喇沙书院）的完整特征：
  - 重视全人发展和领导力
  - 强调团队合作和沟通能力
  - 面试形式：个人面试 + 小组活动
  - 常见话题：课外活动、团队经验、道德判断

#### Scenario: 预置3-5所TOP学校
- **WHEN** 系统初始化完成
- **THEN** 至少包含SPCC、QC、LSC三所学校的完整特征数据
- **AND** 可选包含其他TOP学校（如DGS、DBS等）
- **AND** 所有数据经过验证和格式化

### Requirement: 学校特征查询
系统SHALL提供便捷的学校特征查询功能。

#### Scenario: 按代码查询学校
- **WHEN** 用户通过学校代码（如"SPCC"）查询
- **THEN** 返回该学校的完整特征信息
- **AND** 以结构化格式展示（CLI表格或Web卡片）

#### Scenario: 列出所有学校
- **WHEN** 用户请求查看所有支持的学校
- **THEN** 列出学校代码、全称、简短描述
- **AND** 按字母顺序或热门程度排序

#### Scenario: 搜索学校
- **WHEN** 用户输入学校名称关键词（如"保罗"）
- **THEN** 返回名称匹配的学校列表
- **AND** 支持中文和英文搜索

### Requirement: 学校特征应用
系统SHALL在计划生成和反馈中应用学校特征数据。

#### Scenario: 生成计划时应用学校特征
- **WHEN** AI生成训练计划时用户指定了目标学校
- **THEN** 系统将该学校的重点考查内容作为上下文传递给AI
- **AND** AI生成的计划倾斜到该学校重视的类别（如SPCC增加英文和时事比例）
- **AND** 在计划说明中注明针对性调整

#### Scenario: 生成题目时应用学校特征
- **WHEN** AI生成题目时关联了目标学校
- **THEN** 系统在提示词中包含该学校的常见话题和风格
- **AND** 生成的题目更符合该校面试特点
- **AND** 题目标记为该学校相关

#### Scenario: 反馈中提供学校针对性建议
- **WHEN** AI生成反馈时知道目标学校
- **THEN** 反馈中包含"针对[学校]建议"部分
- **AND** 指出回答如何更好地匹配该校期望
- **AND** 举例该校面试官可能的关注点

### Requirement: 学校特征管理
系统SHALL支持更新和补充学校特征数据。

#### Scenario: 添加新学校
- **WHEN** 用户添加系统未预置的学校
- **THEN** 通过CLI或配置文件输入学校代码、名称和特征
- **AND** 系统验证必填字段并保存
- **AND** 新学校立即可用于计划生成

#### Scenario: 更新学校特征
- **WHEN** 用户发现学校特征数据过时或不准确
- **THEN** 用户可编辑学校的任意字段
- **AND** 系统保存修改历史
- **AND** 更新后的特征应用于新创建的计划和题目

#### Scenario: 标记学校数据来源
- **WHEN** 学校特征数据被录入或更新
- **THEN** 记录数据来源（如"官网资料"、"用户反馈"、"AI推断"）
- **AND** 记录更新时间和更新人
- **AND** 低置信度数据标注"待验证"

### Requirement: 学校对比功能
系统SHALL支持对比多所学校的面试特征。

#### Scenario: 对比两所学校
- **WHEN** 用户请求对比两所学校（如"SPCC vs QC"）
- **THEN** 并排展示两校的关键特征：
  - 重点考查内容对比
  - 面试形式差异
  - 适合的准备策略差异
- **AND** 高亮显示主要区别

#### Scenario: 选择学校建议
- **WHEN** 用户查看多所学校对比
- **THEN** 系统根据学生历史表现（如英文强、逻辑弱）
- **AND** 给出哪些学校更匹配学生优势的建议
- **AND** 提示需要针对性加强的方面
