# Change: 更新項目文檔

## Why

当前 `openspec/project.md` 文檔与項目实际实现狀態不一致：
- 文檔中的功能模块列表不完整，未反映已实现的所有能力
- 技術栈信息需要更新以匹配实际使用的技術
- 項目結构描述需要与实际代碼結构對齐
- 缺少對已归檔变更和当前能力的完整說明
- 需要反映MySQL访問規范、性能優化、計劃-練習聯動等已实现的改進

## What Changes

- **更新項目功能模块列表**：反映所有已实现的能力（訓練計劃、面試練習、AI反馈、題庫管理、進度追踪、學校檔案、係統设置、計劃-練習聯動）
- **更新技術栈信息**：確认实际使用的技術（React 18 + TypeScript + Ant Design, Node.js + Express + TypeScript, MySQL 8.0）
- **更新項目結构**：与实际代碼目錄結构對齐
- **更新能力列表**：列出所有已归檔到specs的能力和待归檔的能力
- **更新開發規范**：添加MySQL访問規范、性能要求、文檔更新要求等
- **更新约束条件**：反映已实现的性能優化、英文題庫語言規范等

## Impact

- 受影响文檔：`openspec/project.md`
- 新增spec：`specs/documentation/spec.md` - 項目文檔管理規范
- 帮助開發者快速了解項目当前狀態和已实现功能
- 確保後续開發基于準確的文檔信息
